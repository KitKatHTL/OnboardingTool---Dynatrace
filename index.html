Onboarding Tool — Projektübersicht

Schnelle statische Demo für GitHub Pages — inklusive Impressum & Urheberrechtshinweis.

Ziel

Das Onboarding Tool unterstützt Bewerbungs- und Onboarding-Prozesse mit Nutzerverwaltung, Checklisten, Automatisierungen (Jira, Miro, Kommunikation) sowie Dashboard & Statistik.

Kernfunktionen
  • NutzerAnlegen von User-Profilen, Statusverwaltung, Archiv
  • ChecklistenGlobale und individuelle Checklisten, Fortschritt sichtbar im Profil
  • AutomatisierungenJira / Miro / Welcome-Messages / Optional OpenAPI
  • DashboardÜbersicht: offene Bewerbungen, Interviews, aktive Onboardings
Technische Basis (Empfohlen)
Backend: Java (Spring Boot) Frontend: React + Material Design DB: PostgreSQL Integrationen: Jira API, Miro API, Slack, OpenAPI
Hinweise

Diese Seite ist eine statische Projektübersicht (HTML/CSS). In der Produktion empfehle ich ein eigenes React-Frontend + Spring Boot Backend wie oben beschrieben.

  1. Repo erstellen: username.github.io oder beliebiges Repo + branch gh-pages
  2. Diese Datei als index.html ins Repo committen
  3. GitHub Pages aktivieren (Settings → Pages → Branch auswählen)
Quick Notes für Entwickler
  • Frontend: Erstelle eine React-App (Vite / Create React App) mit Material UI; die statische Seite ist nur Demo.
  • Backend: Spring Boot Services für Nutzerverwaltung, Checklisten & Integrationen. OAuth 2.0 für API-Integrationen (Jira/Miro/Slack).
  • Security: DSGVO prüfen, Logging & Rollen (Admin / Captain / Buddie / User).
